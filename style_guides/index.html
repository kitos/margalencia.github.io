Зачем проекту style guide

  1. Проект пишется на протяжении трех лет
  2. Несколько поколений разработчиков из 3 городов
  3. Каждый пишет код по своим правилам
  4. Проблемы с шарингом знаний внутри команды
  5. Большое количество UI-багов
  6. Фронтовый код иногда пишут backend-разработчики

Проблемы с HTML

  1. Неоптимальная разметка, избыточность
  2. Не семантическая верстка
  3. Проблемы с доступностью с клавиатуры

Проблемы с CSS

			dd {
				label {color: #E4323E;}
				td:first-child, td.service-name {
					&.missed {
			        	color:#EB0601;
					}
				}
			}

Проблемы с CSS

  1. Большая специфичность селекторов (каскады, как результат злоупотребления препроцессорами)
  2. Селекторы по элементам
  3. Отсутствие переиспользуемых компонентов
  4. Нет единых стилей для типографики, цвета и размеры не вынесены в переменные

Проблемы с JS

  1. Разное форматирование
  2. Не используются best practices
  3. Сильно связан с HTML и CSS
